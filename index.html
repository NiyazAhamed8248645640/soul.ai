How Guardian's Voice works

Part 1 — Voice Trigger

Always-on low-power wake-word detection + speaker verification + acoustic stress classifier (shouting / irregular breathing / high energy) to confirm duress before triggering capture.

Read details

Part 2 — Crash Detection

Sensor fusion (accelerometer, gyroscope, GPS, microphone) to detect high-g impacts. If detected, a 60-second cancel window appears; otherwise the emergency sequence triggers.

Read details

Part 3 — Secure Upload & n8n

Pre-trigger camera ring-buffers + post-trigger recording are encrypted and uploaded to cloud storage via presigned URLs. The app posts a signed JSON payload to an n8n webhook with time-limited media links.

See payload

Key Features

  • On-device voice enrollment and speaker identification
  • Acoustic stress analysis and anti-replay checks
  • Low-power KWS using quantized TFLite or Picovoice/Porcupine
  • Multi-sensor crash detection and 60s cancel protocol
  • Secure, prioritized media upload with presigned URLs and short-lived sharing links
